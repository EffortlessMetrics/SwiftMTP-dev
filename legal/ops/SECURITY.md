# Security Policy

## Reporting Security Issues

**Report to:** security@effortlessmetrics.com
**PGP Key:** [Fingerprint will be provided upon request]
**Response Time:** Within 3 business days

**Do not** open public issues for vulnerabilities.

## Coordinated Disclosure Process

1. **Acknowledgment:** We acknowledge receipt within 3 business days
2. **Assessment:** We assess the issue and determine impact
3. **Communication:** We agree on disclosure timeline with reporter
4. **Fix Development:** We develop and test the fix
5. **Release:** We release the fix and public advisory simultaneously

## Supported Versions

- **Latest:** Full support for latest stable release
- **N-1:** Security fixes backported for previous major version
- **N-2:** Critical security fixes only, upgrade recommended

## Security Updates

- **Critical:** Released within 7 days of fix availability
- **High:** Released within 30 days of fix availability
- **Medium/Low:** Addressed in next scheduled release

## Scope

In scope:
- SwiftMTP core library and CLI
- Transport layer implementations
- File system operations
- Device communication protocols

Out of scope:
- Third-party dependencies (report to respective maintainers)
- Custom modifications
- End-user applications using SwiftMTP

## Recognition

We credit security researchers unless requested otherwise. Credits include:
- Name/handle in release notes and security advisories
- Hall of Fame on our website (if desired)

## Legal Safe Harbor

We consider security research conducted in accordance with this policy to be authorized. We will not initiate legal action against researchers who follow this policy.
